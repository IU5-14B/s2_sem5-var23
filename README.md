# Семинар 5

Проект продолжает предметную область из прошлых семинаров: музей.
В семинаре 5 основное внимание перенесено на собственный шаблонный
контейнер `MyContainer<T>`, внутренний итератор и применение стандартных
алгоритмов C++ к пользовательской коллекции.

В качестве предметной области используются музей, залы и экспонаты:

- `Painting`
- `Sculpture`
- `Artifact`

При этом архитектура из семинара 4 сохранена: в проекте по-прежнему есть
полиморфизм, композиция, агрегация, ассоциация и делегирование.

## Цель семинара 5

В работе реализованы требования задания:

- шаблонный класс `MyContainer<T>` как динамический массив;
- внутренний итератор для собственной коллекции;
- применение `std::find` и `std::sort` к `MyContainer<T>`;
- прикладная задача на предметной области музея;
- демонстрация работы коллекции в реальной программе.

## Что реализовано

### 1. Шаблонный контейнер `MyContainer<T>`

Файл: `include/MyContainer.h`

Контейнер поддерживает:

- динамическое хранение элементов любого типа;
- `push_back`;
- `pop_back`;
- доступ по индексу через `operator[]`;
- `size()`;
- `empty()`;
- копирование и перемещение;
- `begin()` / `end()` для обычного и константного обхода.

Контейнер реализован не через `std::vector`, а вручную, с управлением
памятью через `std::allocator`.

### 2. Внутренний итератор

В `MyContainer<T>` реализован внутренний итератор:

- разыменование `*it`;
- доступ к полям через `it->`;
- префиксный и постфиксный инкремент;
- декремент;
- сравнение итераторов;
- смещение на произвольное число позиций;
- совместимость со стандартными алгоритмами.

Итератор имеет характеристики:

- `iterator_category`
- `value_type`
- `difference_type`
- `pointer`
- `reference`

За счёт этого контейнер можно использовать в алгоритмах стандартной
библиотеки.

### 3. Предметная область музея

Основные классы проекта:

- `MuseumItem` — абстрактный базовый класс музейного объекта;
- `Exhibit` — общий базовый класс экспоната;
- `Painting` — картина;
- `Sculpture` — скульптура;
- `Artifact` — артефакт;
- `Hall` — зал музея;
- `Catalog` — каталог для поиска;
- `Guide` — гид;
- `Visitor` — посетитель;
- `Museum` — основной класс музея.

### 4. Использование `MyContainer<T>` в проекте

Собственный контейнер встроен в предметную область:

- `Museum` хранит залы в `MyContainer<Hall>`;
- `Museum` хранит экспонаты в `MyContainer<std::shared_ptr<MuseumItem>>`;
- `Catalog` выполняет поиск по собственной коллекции;
- прикладная задача строит новый маршрут тоже на `MyContainer`.

## Прикладная задача

Для музея реализован сценарий:

- найти нужный зал через `std::find`;
- собрать короткий маршрут по этому залу;
- отобрать экспонаты, подходящие по условию времени осмотра;
- отсортировать найденные экспонаты по году через `std::sort`.

В программе это показано на примере зала `"Русская живопись"`.

## Где именно используются алгоритмы STL

### `std::find`

Используется для поиска зала в `MyContainer<Hall>`.

Для этого у класса `Hall` реализован `operator==`.

### `std::sort`

Используется для сортировки маршрута из экспонатов в
`MyContainer<std::shared_ptr<MuseumItem>>`.

Сортировка выполняется:

- сначала по году;
- затем по названию, если годы совпадают.

## Что демонстрирует программа

После запуска программа показывает:

- информацию о музее;
- список залов;
- коллекцию экспонатов;
- полиморфную обработку объектов через базовый интерфейс;
- план осмотра;
- поиск зала через `std::find`;
- прикладную задачу семинара 5;
- сортировку маршрута через `std::sort`;
- использование `operator[]` и `pop_back` у `MyContainer<T>`;
- маршрут экскурсии;
- поиск экспоната через каталог;
- пример `typeid` и `dynamic_cast`.
